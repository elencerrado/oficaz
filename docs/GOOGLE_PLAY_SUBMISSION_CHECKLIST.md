## Data Safety & Privacy Compliance

### Required Play Console Declarations

#### 1. **Data Safety Section** (Critical)
**Location**: Google Play Console → App → Data Safety

- [ ] **Personal Data Collection**
  - Declare **what data** the app collects (e.g., user account, time tracking records, contact info from CRM)
  - Specify collection method: account creation, in-app activity, device ID, etc.

- [ ] **Sensitive Permissions Classification**
  - Review `AndroidManifest.xml` for any added permissions:
    - `CAMERA`, `MICROPHONE`, `LOCATION`, `CONTACTS`, `CALENDAR`, `PHONE` → Requires Data Safety declaration
    - Current manifest only declares `INTERNET` → **Low risk profile**

- [ ] **Data Sharing & Retention**
  - Document if data is shared with third parties (Stripe for payments, R2 Object Storage, etc.)
  - Specify retention period (e.g., "until user account deletion")

- [ ] **Data Encryption**
  - Confirm HTTPS/TLS for all backend communication (Express server with security headers)
  - R2 Object Storage: Encrypted in transit ✅
  - Auth token storage: AES-256 encrypted ✅

#### 2. **Content Rating Questionnaire**
**Location**: Google Play Console → App → App Content

- [ ] Complete Google Play Content Rating Questionnaire
  - Violence, Profanity, Adult themes: All likely "None"
  - Business/Financial: **Select "Yes"** (time tracking, HR app)
  - Submit for IARC rating (automatic, ~20 mins)

#### 3. **Privacy Policy**
**Location**: Google Play Console → App → Store Listing

- [ ] **Upload Privacy Policy** (mandatory, Google Play enforces since Nov 2021)
  - **Requirement**: Must be accessible via Privacy Policy link in Store Listing
  - **Content Checklist**:
    - [ ] What personal data is collected
    - [ ] Why it's collected
    - [ ] How users can request data deletion
    - [ ] How you secure data
    - [ ] Third-party sharing (Stripe, R2, email services)
    - [ ] Cookie/tracking policy
    - [ ] Children's privacy compliance (COPPA if applicable)

  - **Recommendation**: Host on `https://oficaz.es/privacy` or similar
  - **Tools**: Use [Privacy Policy Generator](https://app-privacy-policy-generator.com/) as template

## Release Management

### Version & Rollout Strategy

1. **Initial Release (Closed Testing)**
   - Version Code: 1 (set via `OFICAZ_ANDROID_VERSION_CODE=1`)
   - Version Name: "1.0.0"
   - Rollout percentage: 10% (Play Console)
   - Monitor metrics for 1 week

2. **Gradual Rollout to 100%**
   - If no critical crashes after 1 week: increase to 50%
   - After 2 weeks at 50%: increase to 100%
   - Full rollout typically takes 3-4 days to all users

3. **Future Updates**
   - Increment `OFFICAZ_ANDROID_VERSION_CODE` by 1 for each upload
   - Use semantic versioning (e.g., "1.0.1", "1.1.0", "2.0.0")

## Pre-Submission Environment Setup

### Release Build Command

```bash
# Set environment variables for release build
$env:OFICAZ_ANDROID_VERSION_CODE = "1"
$env:OFICAZ_ANDROID_VERSION_NAME = "1.0.0"
$env:OFICAZ_ANDROID_KEYSTORE_PATH = "path/to/keystore.jks"
$env:OFICAZ_ANDROID_KEYSTORE_PASSWORD = "keystore_pw"
$env:OFICAZ_ANDROID_KEY_ALIAS = "alias_name"
$env:OFICAZ_ANDROID_KEY_PASSWORD = "key_pw"

# Build AAB
npm run build
npx cap sync android
cd android
./gradlew.bat bundleRelease

# Output: android/app/build/outputs/bundle/release/app-release.aab
```

### Keystore Generation (if not already done)

```bash
# Generate new keystore (one-time, keep safe!)
keytool -genkey -v -keystore oficaz-release.jks -keyalg RSA -keysize 2048 -validity 10000 -alias oficaz_key

# List keystore contents to verify
keytool -list -v -keystore oficaz-release.jks
```

## Submission Workflow (Google Play Console)

1. **Create App in Play Console** (if not done)
   - Package name: `com.oficaz.app` ✅
   - App icon, store listing metadata

2. **Upload AAB**
   - Play Console → App → Release → Production/Internal Test
   - Select `app-release.aab`
   - Automated analysis runs (~5-10 min)

3. **Review Automated Analysis**
   - Play Console flags any missing policies
   - Fix Data Safety, Privacy Policy, Content Rating if needed

4. **Submit for Review**
   - Final review by Google Play team (~24-48 hours typical)
   - May request additional info on features/permissions

5. **Rollout**
   - Once approved: schedule rollout (10% → 50% → 100%)
   - Monitor crashes/ratings daily first week
